# 🚀 Getting Started

## Prerequisites

Before running this project, make sure you have:

- A modern web browser
- Git installed
- Visual Studio Code (Recommended)
- Live Server Extension (Recommended)

---

## Clone the Repository

```bash
git clone https://github.com/dchaity/dchaity.github.io.git
```

---

## Navigate to the Project Folder

```bash
cd dchaity.github.io
```

---

## Run the Project

### Option 1 — Open Directly

Simply open:

```
index.html
```

in your browser.

---

### Option 2 — VS Code Live Server (Recommended)

1. Open the project in VS Code.
2. Install the **Live Server** extension.
3. Right-click **index.html**.
4. Select **Open with Live Server**.

The website will automatically reload whenever you save changes.

# 🎨 Customization Guide

This portfolio is designed to be easily customizable.

---

## 👤 Change Profile Picture

Replace:

```

assets/profile.jpeg

```

with your own image.

Keep the same filename or update the path inside:

```

index.html

```

---

## 📄 Update Resume / CV

Replace:

```

assets/cv.pdf

```

with your latest CV.

No additional changes are required if the filename remains the same.

---

## ✏️ Edit Personal Information

Open:

```

index.html

```

Update:

- Name
- About Me
- Education
- Skills
- Contact Information
- Social Media Links

---

## 🎨 Change Website Colors

Inside:

```

style.css

```

Locate the root variables.

```css
:root {
    --primary-color: #6c63ff;
    --secondary-color: #8b5cf6;
    --background-color: #0f172a;
    --card-color: #1e293b;
    --text-color: #f8fafc;
}
```

Modify these values to completely change the website's color palette.

---

## 🔤 Change Fonts

Replace the Google Font import inside:

```

index.html

```

or

```

style.css

```

Example:

```css
font-family: 'Poppins', sans-serif;
```

---

## ➕ Add a New Skill

Inside the Skills section, add another skill card.

Example:

```html
<div class="skill-card">
    <i class="fa-solid fa-code"></i>
    <h3>React.js</h3>
    <p>Building modern user interfaces using reusable components.</p>
</div>
```

---

## 🚀 Add a New Project

Duplicate an existing project card and update its content.

Example:

```html
<div class="project-card">

    <img src="assets/project-images/project.jpg" alt="Project">

    <div class="project-content">

        <h3>Project Name</h3>

        <p>
            Short description of the project.
        </p>

        <div class="tech-list">
            <span>HTML</span>
            <span>CSS</span>
            <span>JavaScript</span>
        </div>

        <div class="project-links">
            <a href="#">Live Demo</a>
            <a href="#">GitHub</a>
        </div>

    </div>

</div>
```

# 🔧 Troubleshooting

If you encounter any issues while running or deploying the portfolio, refer to the table below.

| Issue | Possible Solution |
|--------|-------------------|
| Images are not displaying | Verify that the image paths are correct and the `assets` folder exists. |
| CSS or JavaScript not loading | Check file names, paths, and browser cache. |
| Contact form is not working | Make sure your Formspree Form ID is correctly configured. |
| CV download fails | Confirm that `assets/cv.pdf` exists and the filename matches the link. |
| GitHub Pages is not updating | Wait a few minutes after pushing or verify **Settings → Pages** is configured correctly. |
| Dark mode preference is not saved | Ensure Local Storage is enabled in your browser. |
| Icons are missing | Verify the Font Awesome CDN link is included in `index.html`. |
| Typing animation is not working | Check that the required JavaScript is loaded correctly. |

# 🚀 Deployment

## GitHub Pages

This portfolio is hosted using **GitHub Pages**.

### Deployment Steps

1. Push your latest code to the **main** branch.
2. Open your GitHub repository.
3. Go to **Settings → Pages**.
4. Under **Source**, select:

```
Deploy from a branch
```

5. Choose:

```
Branch: main
Folder: / (root)
```

6. Save the changes.

GitHub will automatically build and publish the website.
